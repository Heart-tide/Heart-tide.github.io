如何度过一个愉快的暑假？

2022-06-22
1.7k- 6m- -
生活日记-假期CS:APP西方哲学史

在最后一场考试后……

我听着小曲，漫游在渐渐入夜的南大校园中，想起来那场重要考试前的那晚，似乎也与今个的夜色并无二致。

我还没有到可以坦然说出“那场考试其实也没有什么”的淡泊心境来，不过比起那些无意识般度过的日日夜夜，一年过去，我也算是抓到了一点心灵的形貌。

这够吗？自我意识的萌发。

风在动，风不回应我。骑车的女人经过。

我或许站着干想也得不到什么答案。所以——

前进吧！留心多思。

毕竟，我们总是太过急躁地去产出一个答案，却轻视了其前置的证明与后续实践的重要性。

干点什么呢？

很多（几乎100%的）时候，当面对大把的可支配时间时，我会做好万全的计划！然后，把这些计划忘掉，按本能进行每一天的生活。

这次，我开了这个贴来记录我的生命。

一是起到记录的功能，二是朋友间的分享，三是也顺便充实一下我的博客。

既然是我的小站嘛，也没有什么拘束，总之：

爷就是这儿的主人！全都我说了算。

所以也没有什么严肃的既有人设，我就“欻欻”两声劈开了藩篱，把这个贴给建出来了。

那么，先看看这个假期我要干点什么吧！

都在这里了，它们也是我轻装简行意志的承继：

物理上的质量很小，但心灵上的密度，一点也不小。

我希望这也是我今后的生存状态。

那么，下面——

我的假期开始啦！！！

可一书店

下午同朋友去了可一书店，那里气氛很好，静雅闲适，更协窗外雨落，友人作伴，眼前哲思亦有趣。可惜雨下得大而急，未能全身而退，可算瑕不掩瑜。

在自然哲学家还在以有形质料为万物本原、各执一词而争论时，赫拉克利特的思想已经超越时代，达到了“对辩证唯物主义原则的绝妙说明”之高度。

运动的普遍性，事物的质与度（logos），普遍规律，变化源于对立统一，以及如何认识被隐藏的事物本性的问题，很难想象它们早在马哲诞生两千多年前，就已具有其雏形。

这使我第一次切实地感受到欧陆哲学的继承发展。之前那种想切入进去、直接学习语言哲学的想法，可是太天真了。

当然总体来讲，自然哲学仍然摆脱不了一个绝大的、感性的、有形的认知世界。

这点苏格拉底已有批判，于是他把论辩的底色引入到了理性与公民德性的空间之中，开创了古希腊哲学的新纪元。

而其继者，柏拉图，对理念世界和可感世界的分野，思想不可不谓不深，更是使我啧啧称奇、连连惊叹的。

然则值得一提的是，苏格拉底死后，他的弟子们所形成的各个小苏格拉底学派无一发挥了苏格拉底老先生的本意。

  • 苏格拉底从不为真理下绝对的定义，而麦加拉学派却开历史倒车，将其与艾利亚学派那个绝大唯一不动的“存在”相等同。

  • 库兰尼学派把“认识论的感受与伦理学的感受混为一谈”，片面追求感官上的享乐。

  • 犬儒学派与库兰尼学派相反，他自封自闭，放弃任何对抽象的理念论的认识，以节制、禁欲为美德。

我何尝没有经历过此三者的思想路径呢？可叹可叹耶！

datalab

经过充分的摸鱼后，我终于来到了 CS:APP 的第一个实验 datalab，这里选几道有意思的题咱瞧瞧：

数据的 attributes

  1. 检验 x 是否为 Tmax，只允许使用 ! ~ & ^ | +
int isTmax(int x)
{
    // 从运算性质方面考察，有且仅有 Tmax 和 -1 满足等式 2 * x == -2
    return (!(x + 2 + x)) & !!(x + 1);
}
  1. 实现三目运算符，只允许使用 ! ~ & ^ | + << >>
int conditional(int x, int y, int z)
{
    // 首先把 x 映射到 {0,1} 上，然后再把 {0,1} 映射到 {0,~0} 上，作掩码
    y = ((!!x) << 31 >> 31) & y;
    z = ((!x) << 31 >> 31) & z;
    return y | z;
}
  1. 实现逻辑非，只允许使用 ~ & ^ | + << >>
int logicalNeg(int x)
{
    // 从表示形式方面考察，0 具有这样的独特性质：+0 和 -0 的符号位均为 0
    // 所以取出符号位后用 1 减去它即可
    return 2 + ~(((x | (~x + 1)) >> 31) & 1);
}

这些运算符各司其职：

! 负责映射，& 用于取出特定位，^ 用于检验全部位，| 用于拼接，而 << >> 和 ~ 最擅长造字面量

在仅对一位布尔值进行操作时，& 与 &&，| 与 || 是完全等效的。

此外，移位操作和 x*2 与 x/2 的对应性也值得留意。

分类与分步

比起上面的“思维训练”，还是考察程序员之基本功的相关题目更得我意：

  1. 实现小于等于的判断，只允许使用 ! ~ & ^ | + << >>
int isLessOrEqual(int x, int y)
{
    int x_is_positive = !(x & (1 << 31));
    int y_is_positive = !(y & (1 << 31));

    // 情况 1，x>0 && y<0 时返回 0，同时 y==Tmin 的情况在这里被排除掉了
    int case_1 = (!x_is_positive) | y_is_positive;

    // 情况 2，x<0 && y>0 时返回 1
    int case_2 = (!x_is_positive) & y_is_positive;

    // 情况 3，x 与 y 同号，此时不存在溢出问题
    int minus_y = ~y + 1;
    int x_minus_y = x + minus_y;
    int x_minus_y_is_negative = !!(x_minus_y & (1 << 31));
    int x_minus_y_is_zero = !x_minus_y;

    // 召唤神龙！
    return case_1 & (case_2 | x_minus_y_is_negative | x_minus_y_is_zero);
}
  1. 实现 float 到 int 的强制类型转换，不允许使用任何函数
int floatFloat2Int(unsigned uf)
{
    // 取出各个功能位，并做预处理
    unsigned sign = uf >> 31;
    int e = (uf << 1 >> 24) - 127;
    unsigned frac = uf << 9 >> 9;
    unsigned total_frac;
    unsigned n;

    // 排除由于阶码过大过小导致 int 装不下的情况
    if (e >= 31) {
        return 1 << 31;
    } else if (e < 0) {
        return 0;
    }

    // 移动小数点
    total_frac = (frac + (1 << 22));
    n = 0;
    if (e <= 22) {
        n = total_frac >> (22 - e);
    } else {
        n = total_frac << (e - 22);
    }

    // 添加正负号
    if (sign == 0) {
        return n;
    } else {
        return -n;
    }
}

有没有发现，IEEE 规定下的浮点数更像是一个多部件的拼合体？
